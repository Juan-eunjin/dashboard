# dashboard
Frontend 코드

사용 기술 : React + typescript



### 개발 진행 팁 (React 중심)
컴포넌트 중심 사고: 대시보드에 들어가는 '진행률 차트', '담당자별 업무량' 등을 각각의 Component 파일로 분리하세요. 그래야 나중에 상세 화면에서도 특정 차트만 재사용하기 편합니다.
데이터 가공 위치: 백엔드에서 가급적 계산이 끝난 수치(%)를 주는 것이 좋지만, 프론트엔드에서도 특정 조건에 따라 데이터를 필터링하는 로직(useMemo 등 활용)이 필요할 수 있습니다.
상세 화면의 연결성: 사용자가 대시보드 그래프에서 '지연됨' 영역을 클릭하면, 상세 화면으로 넘어가면서 자동으로 '지연된 이슈'만 필터링되어 보여주는 인터랙션을 고려해 보세요.
